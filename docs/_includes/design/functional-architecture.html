Description

Functionally, the robot decomposes into operator/mission input, safety supervision, state estimation, local scene representation, body-motion command synthesis, gait phase scheduling, phase-to-RPM routing, low-level motor execution, and telemetry. The final stack keeps the same high-level functional decomposition but implements it with a concrete ROS/ESP split.

Local scene representation comes from the D415, ORB-SLAM2 CUDA, virtual scan extraction, and slam_toolbox. Body state comes from fused ORB odometry, Klann kinematic odometry, and IMU yaw-rate/attitude information where validated. Gait phase scheduling preserves three-foot support through alternating tripod relationships, while startup sequencing moves a known boot pose toward a contact pose before controller commands are trusted.
