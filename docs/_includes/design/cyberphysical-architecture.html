Description

The final system is split into a Jetson Nano high-level autonomy computer and an ESP32 low-level motor/power controller. ORB-SLAM2 CUDA and the RealSense D415 process run directly on the Jetson host. ROS 2 Jazzy runs inside the container and owns state estimation, Klann kinematic odometry, odometry fusion, the ORB-SLAM UDP bridge, virtual scan generation, slam_toolbox, Nav2 bring-up, visualization, and the browser operator interface.

The ESP32 owns deterministic power and actuator I/O: precharge, contactor closure, shared servo-enable control, motor-driver configuration, torque/current limits, changed-only RPM writes, strict six-drive raw-encoder sweeps, and micro-ROS publication of compact encoder/timing telemetry.

Command flow is browser teleop or Nav2 → /hexapod/cmd_vel_selected → tripod/MPPI gait generation → /hexapod/phase_cmd → phase-to-RPM routing → /motor_rpm_cmd → ESP32 RS485 writes. Feedback flow is ESP32 /motor_output_odom → Jetson motor-state aggregation → continuous crank phase → Klann body-state estimation → fused /odom and odom → base_link TF.
